# 使用教程：科研通半自动化应助Chrome浏览器插件

本文档是 AbleSci Assist Guard 的中文详细教程。

## 0. 名称和适用范围

- 中文定位：科研通半自动化应助Chrome浏览器插件
- 扩展名称：AbleSci Assist Guard
- 当前适用：Chrome 浏览器

## 1. 插件是什么

AbleSci Assist Guard 是一个 Chrome 浏览器插件，用于辅助科研通 / AbleSci 文献互助应助流程。本项目针对科研通文献互助平台打造，也就是：

```text
https://www.ablesci.com/
```

它主要帮助你完成这些重复操作：

- 打开科研通求助列表。
- 刷新 Elsevier 求助中页面。
- 接本页最后一条可见求助。
- 抓取求助标题、DOI 和链接。
- 打开 DOI / 出版商页面。
- 自动点击明确的 `View PDF`。
- 在你手动下载 PDF 后，自动复制最新 PDF 的本地路径。

它不是完全自动化工具。下载、上传和最终提交仍然需要你自己确认。使用者需要自备有权限的账号，并确保所有文献访问、下载、分享和上传行为合法合规。

## 2. 浏览器要求

当前版本适配：

- Chrome 浏览器

## 3. 安装前准备

你需要先准备：

1. Chrome 浏览器。
2. 科研通 / AbleSci 账号。
3. 自备有权限的学校、机构或出版商访问账号。
4. 已经登录的学校数据库或出版商访问入口。
5. 下载好的插件项目文件夹。

插件不需要：

- Docker。
- Python 环境。
- Node.js 环境。
- 本地服务器。

## 4. 安装步骤

### 第一步：下载项目

如果你从 GitHub 下载：

1. 打开项目仓库页面。
2. 点击绿色 `Code` 按钮。
3. 点击 `Download ZIP`。
4. 解压 ZIP 文件。

解压后，文件夹内应该能看到：

```text
manifest.json
background.js
content.js
content.css
popup.html
popup.css
```

注意：你要选择包含 `manifest.json` 的那一层文件夹。

### 第二步：打开 Chrome 扩展管理页

在 Chrome 地址栏输入：

```text
chrome://extensions/
```

然后回车。

### 第三步：开启开发者模式

在扩展管理页右上角打开：

```text
开发者模式 / Developer mode
```

### 第四步：加载插件

点击：

```text
加载已解压的扩展程序 / Load unpacked
```

选择插件文件夹。

如果加载成功，你会看到扩展列表中出现：

```text
AbleSci Assist Guard
```

### 第五步：刷新科研通页面

打开或刷新：

```text
https://www.ablesci.com/
```

页面右下角应该会出现插件面板。

## 5. 首次使用前检查

建议你先确认：

1. 科研通已经登录。
2. 自备的有权限账号可以正常使用。
3. 学校数据库入口已经登录。
4. 出版商页面如果需要机构认证，已经能正常访问。
5. Chrome 可以正常下载 PDF。
6. Chrome 下载路径是你能找到的位置。

## 6. 一键 Elsevier 流程

这是最常用的流程。

### 操作步骤

1. 打开科研通页面。
2. 找到右下角 `AbleSci Assist Guard` 面板。
3. 点击 `Elsevier求助中`。

插件会自动执行：

1. 跳转到 Elsevier 求助中列表。
2. 给列表地址添加临时时间戳，强制刷新。
3. 等待列表加载。
4. 自动接本页最后一条可见求助。
5. 进入求助详情页。
6. 抓取标题、DOI 和链接。
7. 打开 DOI / 出版商页面。
8. 如果出现明确的 `View PDF`，自动点击进入 PDF 查看页。

### 为什么接最后一条

第一页最靠前的求助通常竞争很激烈，可能已经有人处理，或者是置顶的困难求助。接本页最后一条可以减少抢单失败和重复处理的概率。

## 7. 扫描列表流程

如果你想自己从当前页挑选一条，可以用扫描列表。

### 操作步骤

1. 打开科研通求助列表页。
2. 点击插件面板中的 `扫描列表`。
3. 插件会列出当前页面可见的求助。
4. 找到你想处理的一条。
5. 点击该条下面的 `打开求助`。

之后会进入和一键 Elsevier 相同的流程。

## 8. PDF 页面和下载路径复制

当插件自动点击 `View PDF` 后，通常会进入 ScienceDirect PDF 查看页。

你需要：

1. 在 PDF 查看页中，手动点击右上角下载按钮。
2. 等待 Chrome 下载完成。
3. 插件会自动检测这次新下载的 PDF。
4. 插件会把 PDF 的本地路径复制到剪贴板。

然后：

1. 回到科研通求助详情页。
2. 找到上传区域。
3. 点击 `浏览文件`。
4. 在文件名输入框中粘贴刚刚复制的路径。
5. 选择文件。
6. 自己检查文件是否正确。
7. 手动提交。

## 9. 为什么不自动上传和提交

上传文件和提交应助会把文件发送到第三方平台，这是有外部影响的操作。

为了避免误传、错传或未经检查的提交，插件只辅助到“复制路径”和“高亮上传区域”，最终上传和提交仍由用户自己完成。

## 10. 为什么不自动点击浏览器 PDF 下载按钮

Chrome PDF 查看器的下载按钮属于浏览器内置界面，不一定是网页 DOM，扩展不一定能稳定控制。

此外，自动下载订阅全文可能涉及访问权限和合规边界。因此插件采用更稳妥的方式：

```text
用户手动点击下载 -> 插件自动检测下载完成 -> 插件自动复制路径
```

## 11. 常见问题

### 页面右下角没有插件面板

请检查：

1. 插件是否已启用。
2. 当前页面是否是科研通、ScienceDirect 或 ScienceDirect PDF 页面。
3. 是否刷新过页面。
4. 是否选中了正确的插件文件夹。

### 点击 Elsevier 后还是旧求助

当前版本会给 Elsevier 列表加临时时间戳，强制刷新列表。如果仍然异常，可以手动刷新页面后再点一次。

### 没有识别到 View PDF

可能原因：

1. 当前账号没有访问权限。
2. 出版商页面还没加载完成。
3. 页面需要机构登录或验证码。
4. 出版商页面按钮文字不是标准 `View PDF`。

你可以完成验证后点击 `继续当前流程`。

### 下载完成后没有复制路径

请检查：

1. Chrome 是否真的完成下载。
2. 插件是否有 `downloads` 权限。
3. 插件是否有 `clipboardWrite` 权限。
4. 是否刷新过插件。

也可以回到科研通页面，手动点击 `锁定并复制最近PDF`。
